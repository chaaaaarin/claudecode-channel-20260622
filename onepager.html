神連携の仕組み（コマンド完全解説）

① Claude Code → Claude Design（デザインシステムをアップロード）
  • 1Claude Codeでデザインシステムを作成（Storybookフォーマットが標準）
  • 2プロジェクトフォルダ内で /designlogin を実行
  • 3表示されたURLをブラウザに貼付 → 承認 → 認証コードをClaude Codeに返す
  • 4/designsync でデザインシステムをClaude Designにアップロード
  • 5Claude Design上でデザインシステムが反映され、ブランド統一で量産可能に
⚠️ /designsync はデザインシステム形式のコンテンツのみ対象。所要時間約30分・約10万トークン消費（🔶要確認）。エラー時は /designlogin で再ログイン。
② Claude Design → Claude Code（デザインを実装に引き渡し）
  • 1Claude Designで「Share」→「Send」→「Claude Code」ボタンでコードを取得
  • 2Claude Codeにコードを貼付 → プロジェクトフォルダにデータ出力
  • 3そのままClaude Codeで実装継続 → Vercelデプロイまで完結
③ ローカル↔クラウドの双方向同期
  • ローカルで編集した→「クラウドにプッシュして」と自然言語で指示
  • クラウドで編集した→「クラウドの内容をローカルに取り込んで」と自然言語で指示

FAQ・注意点

無料プランで使えますか？
現状はPro/Max/Team/Enterpriseの有料プランのみ。無料プランでの利用は不可の模様。（✅ 要公式確認）
/designsync の初期設定が難しそう
難易度は高めです。Claude Codeと対話しながら進めるか、公式ドキュメントを参照してください。エラー時は /designlogin で再ログイン。（🔶）
Figmaは不要になる？
エンタープライズチームはFigmaから離れていない、という観測が多い。Claude Designは「速く作る」ツールとして使い分けるのが現実的。
ProプランでもトークンはOK？
共通枠になり使いやすくなりましたが、/designsync には約10万トークン消費するとの情報あり（⚠️要確認）。Proでも制限に達する可能性はあります。
